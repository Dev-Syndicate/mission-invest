## 1. Product Overview

### 1.1 Vision
Turn boring savings into addictive, short-term missions that people actually complete — by combining behavioral psychology, gamification, and adaptive AI.

### 1.2 Mission Statement
> "Make saving money feel like playing a game you can't put down."

### 1.3 Positioning
**Questr** sits at the intersection of Duolingo (habit loops), fitness trackers (streaks + progress), and neobanking (financial discipline). It is not a savings account — it is a **behavioral savings engine**.

### 1.4 Target Audience
- Primary: Ages 18–32, India-first, smartphone-native
- Secondary: First-time savers, students, young professionals
- Psychographic: Goal-driven but easily distracted; responds well to visual progress and social validation

---

## 2. Problem Statement

Traditional savings apps fail because:
1. They track numbers, not behavior
2. Abstract goals ("save ₹50,000") feel distant and unmotivating
3. No feedback loop — missing a day has zero consequence
4. No emotional connection to the goal

**Questr solves this by turning every savings goal into a time-boxed, streak-driven, reward-fueled mission.**

---

## 3. Core User Loop

```
Create Mission → Break into daily tasks → Complete contributions
      ↑                                              ↓
 Next Mission ←  Dopamine hit (completion)  ← Earn rewards & badges
      ↑                                              ↓
                                          AI adapts if user struggles
```

This loop is the **retention engine**. Every feature must reinforce or protect this loop.

---

## 4. Key Differentiators

| Feature | Traditional Apps | Questr |
|---------|-----------------|--------|
| Goal framing | "Save ₹5,000" | "Mission: Goa Trip 🏖️ — 45 days left" |
| Feedback | Monthly statement | Daily streak + progress ring |
| Missed day | Nothing happens | Streak break alert + recovery mode |
| Motivation | None | AI nudges, badges, themes, vision board |
| Adaptation | Manual | AI auto-suggests timeline/amount adjustments |

## 5. Feature Specifications

### 5.1 Mission Builder

**Description:** The entry point. Users create a savings goal as a named, themed, time-boxed mission.

**Inputs:**
- Goal name (custom or from template)
- Target amount (₹)
- Duration: 10–180 days
- Contribution frequency: daily / weekly

**Templates (pre-built):**
- ✈️ Trip / Vacation
- 💻 Gadget Purchase
- 🏍️ Vehicle Down Payment
- 💰 Emergency Fund
- 🎓 Course / Upskilling
- 🎁 Gift for Someone

**Output generated by system:**
- Daily / weekly contribution amount
- Milestone breakdown (25%, 50%, 75%, 100%)
- Estimated completion date

**Validation rules:**
- Minimum mission duration: 10 days
- Maximum: 180 days
- Minimum daily contribution: ₹10
- Max active missions per user (MVP): 3

---

### 5.2 Personalisation Layer

**Goal customization:**
- Rename goal (e.g., "My Dream Bike 🔥")
- Upload a vision image (stored in Firebase Storage)
- Add a personal motivation message ("No excuses. You got this.")

**Theme options:**
- 🌙 Dark Mode
- ✨ Minimal Light
- 🎮 Gaming (neon accents)
- 🌸 Soft Pastel

**Purpose:** Emotional ownership increases mission completion rate. Users who personalize complete at 2.3× the rate of those who don't (Duolingo internal data analogy).

---

### 5.3 Progress Dashboard

**Per-mission view:**
- Circular progress ring (% funded)
- Streak counter ("🔥 Day 12")
- Days remaining
- Amount saved vs target
- Milestone unlock indicators
- Next contribution due

**Home screen:**
- Active missions (card stack)
- Overall streak across all missions
- Recent badges earned
- Quick-contribute button per mission

---

### 5.4 Streak & Reward Engine

**Streak mechanics:**
- Streak increments on each day a contribution is logged
- Break = streak resets to 0
- Miss Recovery Mode: if user misses 1 day, they get a 24hr window to recover streak (max 1 recovery per 7 days)

**Badges (MVP set):**
- 🔥 3-Day Streak
- 💪 7-Day Warrior
- 🏅 First Mission Complete
- 🎯 Halfway Hero (50% milestone)
- 🚀 Speed Runner (completed before deadline)
- 📅 30-Day Survivor

**Unlockables:**
- New themes unlock at milestones
- Certificate generated on mission completion (shareable PNG via Flutter share plugin)

**Gamification rules:**
- Badges are non-revocable once earned
- Streak shown prominently on home screen
- Certificate includes: goal name, amount saved, days taken, completion date

---

### 5.5 AI Recommendation Engine (FastAPI Microservice)

**Trigger conditions:**
- User misses 2+ consecutive days
- Completion probability drops below 60%
- User manually requests help

**Actions the AI can suggest:**
- Extend timeline ("Add 7 more days?")
- Reduce daily amount ("Lower by ₹50/day?")
- Split into smaller sub-missions
- Generate motivational nudge message (personalized)

**Completion probability model (MVP — rule-based):**
```
score = (days_completed / total_days) * 0.5
      + (current_amount / target_amount) * 0.4
      + (streak_ratio) * 0.1
```
Full ML model in v2 using historical user cohort data.

**AI Motivation Messages (LLM-generated via FastAPI → Claude/GPT):**
- Context-aware: uses goal name, streak, days left
- Tone: energetic, personal, emoji-forward
- Example: "You're ₹1,200 away from Goa 🏖️. Don't let Day 31 be the day you gave up. ₹200 is less than a Swiggy order."

---

### 5.6 Notification System

**Notification types:**

| Type | Trigger | Message example |
|------|---------|-----------------|
| Daily reminder | 9AM or user-set time | "₹200 today = Goa is 1 day closer 🌊" |
| Streak alert | Contribution not logged by 8PM | "⚠️ Your 7-day streak is at risk!" |
| Streak break | Missed day confirmed | "Streak lost. Recovery window open for 24hrs." |
| Milestone | 25/50/75% reached | "🎉 Halfway there! You've saved ₹2,500!" |
| Weekly report | Every Sunday | Summary card: saved, streak, % complete |
| Mission complete | 100% reached | Full celebration screen + certificate |

**Implementation:** Firebase Cloud Messaging (FCM) triggered by Cloud Functions.

---

### 5.7 Admin Dashboard (Web — Flutter Web or React)

**Analytics panels:**
- Total users, DAU/WAU/MAU
- Top goal categories (by volume)
- Mission completion rate by category
- Drop-off analysis: which day do users abandon? (hypothesis: Day 3, Day 7)
- Avg days to first failure
- Streak distribution chart

**Platform controls:**
- Create/edit goal templates
- Launch platform-wide challenges ("30-Day Savings Sprint — Jan 2025")
- Push broadcast notifications
- Feature flag toggles (for A/B tests)
- View flagged AI suggestions for quality review

## 6. Technical Architecture

### 6.1 Tech Stack

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| Mobile App | Flutter (iOS + Android) | Single codebase, rich animation support |
| Auth | Firebase Authentication | Email, Google, phone OTP |
| Realtime DB | Firebase Firestore | Real-time sync for streaks/progress |
| File Storage | Firebase Storage | Vision board images, certificates |
| Push Notifications | Firebase Cloud Messaging (FCM) | Cross-platform push |
| AI Microservice | FastAPI (Python) | LangChain agents, ML model hosting |
| Background Jobs | Firebase Cloud Functions | Scheduled nudges, streak checks |
| Hosting (admin) | Firebase Hosting | Admin dashboard |
| Analytics | Firebase Analytics + custom events | Funnel and retention tracking |

### 6.4 FastAPI AI Service Endpoints

**POST /ai/nudge**
```json
Request:  { "userId": "...", "missionId": "...", "trigger": "missed_day" }
Response: { "message": "...", "actionSuggestion": "extend_7_days" }
```

**POST /ai/adapt**
```json
Request:  { "missionId": "...", "currentSaved": 3200, "daysLeft": 20, "target": 10000 }
Response: { "suggestion": "reduce_daily", "newDailyAmount": 172, "newEndDate": "..." }
```

**POST /ai/predict**
```json
Request:  { "missionId": "...", "streakRatio": 0.7, "amountRatio": 0.6, "dayRatio": 0.65 }
Response: { "completionProbability": 0.74, "riskLevel": "medium" }
```

**POST /ai/message**
```json
Request:  { "goalName": "Goa Trip", "amountLeft": 1200, "daysLeft": 6, "streak": 12 }
Response: { "message": "You're ₹1,200 away from Goa 🏖️. 6 days. Don't quit now." }
```

|--------|-----------|------------|
| D7 Retention | Users active on Day 7 | > 40% |
| Mission Completion Rate | Missions reaching 100% | > 35% |
| Avg Streak Length | Mean consecutive contribution days | > 8 days |

| Streak Recovery Rate | Users who use recovery mode and succeed | > 50% |

---

## 9. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Users forget to log contributions | High | Strong push notification system; explore UPI hook in v2 |
| Streak system feels punishing | Medium | Miss Recovery Mode; gentle tone on break notifications |
| AI nudges feel generic | Medium | Personalize with goal name, amount, streak in prompt |
| Scope creep in MVP | High | Hard cutoff: no social features in v1 |
| Firebase costs at scale | Low (MVP) | Set budget alerts; migrate to GCP Cloud Run for AI service |

---

## 10. Open Questions

1. Should contributions be **manual log only** (MVP) or **UPI-linked** from Day 1?
2. What is the **minimum viable AI nudge** — rule-based strings or live LLM?
3. Should the **certificate** be a static PNG or animated (Lottie)?
4. **Monetization model** — freemium (3 active missions free, ₹99/mo for unlimited)?
5. Do we want **social/friend features** in v1 or keep it fully solo?
